**User-Provided Data Integration section** (KWRD-04):

Heading: `## User-Provided Data Integration`

This section handles the case where the user's keyword input includes volume and/or difficulty scores from their paid tools (AppTweak, Sensor Tower, data.ai, etc.).

Rules:
1. **Preserve exactly** -- never modify, round, or re-estimate user-provided volume/difficulty numbers. Display them in their original form.
2. **Label as [USER-PROVIDED]** -- distinct from OBSERVED (live data we fetched) and ESTIMATED (Claude's analysis). User-provided data comes from the user's paid tools and we trust it as-is.
3. **Display alongside Claude's analysis** -- the keyword table should show both Claude's relevance score AND the user's volume/difficulty if provided.

Enhanced table format when user provides scores:
```
| Keyword | Relevance | Volume | Difficulty | Source |
|---------|-----------|--------|------------|--------|
| meditation | 10/10 [ESTIMATED] | 62 [USER-PROVIDED] | 43 [USER-PROVIDED] | Seed |
| mindfulness | 9/10 [ESTIMATED] | -- | -- | Expansion |
```

When user does NOT provide volume/difficulty:
- Do NOT fabricate numbers. Do NOT estimate numeric volume or difficulty.
- Instead, the Popularity Signals section below provides directional guidance.
- The table omits Volume and Difficulty columns entirely (no empty columns, no dashes).

**Popularity Signals section** (KWRD-04 continued):

Heading: `## Popularity Signals`

When the user has NOT provided volume/difficulty data, provide directional popularity signals using Apple autocomplete as a proxy. This is NOT a substitute for paid tool data -- it is a free, directional signal.

Method:
1. For each keyword (top 20 by relevance score), use WebSearch to check if the term appears in Apple App Store autocomplete suggestions by searching: `{keyword} site:apps.apple.com`
2. Classify each keyword into a popularity tier:

| Tier | Signal | Interpretation |
|------|--------|----------------|
| High | Appears in autocomplete, multiple top apps target this term | Likely high search volume -- competitive |
| Medium | Appears in autocomplete or found in several app titles/subtitles | Moderate search interest |
| Low | Not in autocomplete, few apps target this term | Niche -- lower volume but potentially lower competition |
| Unknown | Could not determine from available signals | Recommend user check paid tools |

3. Display as an additional column in the keyword table: `Popularity: High/Medium/Low/Unknown [ESTIMATED]`

Important caveats (include these in the section):
- "Popularity tiers are directional estimates based on Apple autocomplete presence and competitor analysis. They are NOT search volume numbers."
- "For accurate volume and difficulty data, use a paid ASO tool (AppTweak, Sensor Tower, data.ai) and re-run /aso:keywords with your data file."
- Reference the Data Integrity directive in rules/aso-domain.md.

If the user DID provide volume/difficulty data, skip this section entirely and note: "Popularity signals skipped -- using your provided volume/difficulty data."

**Intent Grouping section** (KWRD-05):

Heading: `## Intent Grouping`

Group ALL keywords (seed + expanded) into intent categories. Every keyword must appear in exactly one group.

| Intent Category | Definition | Example Keywords |
|-----------------|------------|-----------------|
| Navigational | User searching for a specific app by name or brand | "spotify", "calm app", "headspace" |
| Feature-Seeking | User searching for a specific capability or function | "sleep tracker", "guided meditation", "breathing exercises" |
| Problem-Solving | User searching for a solution to a problem | "can't sleep", "reduce anxiety", "focus at work" |
| Comparison | User evaluating options or alternatives | "best meditation app", "calm vs headspace", "free yoga app" |
| Category | User browsing a general app category | "wellness app", "health and fitness", "mindfulness apps" |

For each group, list the keywords sorted by relevance score (highest first). Include the relevance score next to each keyword.

Format:
```
### Feature-Seeking
| Keyword | Relevance | {Volume if provided} |
|---------|-----------|----------------------|
| guided meditation | 9/10 | {value or --} |
| sleep tracker | 8/10 | {value or --} |
```

After all groups, provide a **Group Summary**:
```
Intent Distribution: Feature-Seeking ({N}), Problem-Solving ({N}), Category ({N}), Comparison ({N}), Navigational ({N})
```

Note which intent group has the most keywords and which has gaps: "Feature-seeking keywords dominate your list. Consider expanding problem-solving terms -- users searching for solutions ('how to sleep better') often have higher install intent."
