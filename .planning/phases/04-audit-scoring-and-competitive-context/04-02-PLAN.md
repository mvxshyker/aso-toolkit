Two changes to commands/aso/audit.md implementing AUDT-09 and completing the Phase 4 audit structure:

**Change 1: Add `## Prioritized Recommendations` section**

Insert AFTER the "Competitive Context" section (from Task 1) and BEFORE the "Analysis Summary" section. This section implements AUDT-09.

Content:

1. **Impact Level Definitions:**
   ```
   | Impact | Definition | Action Timeline |
   |--------|------------|-----------------|
   | Critical | Violates store policy or causes listing rejection/suspension | Fix immediately |
   | High | Directly impacts search ranking or conversion rate | Fix this week |
   | Medium | Missed optimization opportunity with moderate impact | Fix this month |
   | Low | Minor improvement, nice-to-have | Backlog |
   ```

2. **Recommendation Synthesis Instructions:**
   - "Review every finding from the Title Analysis, Subtitle/Short Description Analysis, Description Analysis, Keyword Field Guidance (iOS), Rating and Review Summary, ASO Score, and Competitive Context sections."
   - "For EACH finding that identifies an issue, gap, or opportunity, create a recommendation entry with:"
     - **Finding**: What was observed or assessed (reference the section it came from)
     - **Impact**: Critical, High, Medium, or Low (using the definitions above)
     - **Recommendation**: A specific, actionable instruction -- not generic advice. Say exactly what to change and why.
     - **Expected Effect**: What improvement the user should expect (e.g., "Improves title keyword coverage, directly boosting search ranking signal")

3. **Output Format:**
   ```
   ### Recommendations (ordered by impact)

   **1. [CRITICAL/HIGH/MEDIUM/LOW] {Brief title}**
   - Finding: {What was found, from which section}
   - Action: {Specific instruction -- what exactly to do}
   - Expected effect: {What this fixes or improves}

   **2. [CRITICAL/HIGH/MEDIUM/LOW] {Brief title}**
   ...
   ```

4. **Ordering Rule:**
   - "Sort recommendations by impact level: Critical first, then High, then Medium, then Low."
   - "Within the same impact level, prioritize Metadata findings over Visibility over Conversion (matching the scoring rubric weight order from rules/aso-domain.md)."

5. **Minimum Recommendations:**
   - "Every audit must produce at least 3 recommendations. If the listing is exceptionally well-optimized, identify refinement opportunities at the Medium or Low level."

6. **Actionability Standard:**
   - "Each recommendation must be specific enough that the user can implement it without further research."
   - BAD: "Improve your title" -- too vague
   - GOOD: "Add your primary keyword 'meditation' to the first position in your title. Current title 'Calm - Sleep & Relax' could become 'Calm - Meditation & Sleep' to front-load the highest-value keyword."
   - BAD: "Get more reviews" -- not actionable
   - GOOD: "Your review count (47) is well below the category average. Implement an in-app review prompt using StoreKit (iOS) or Google Play In-App Review API after a positive user action (e.g., completing a session)."

**Change 2: Update the Analysis Summary section**

Find the existing "## Analysis Summary" section and update it:

1. **Update the Quick Stats Table** to add a row for the ASO Score:
   - After the existing field rows (Title, Subtitle, Description), add:
   ```
   | **ASO Score** | {score}/100 | {grade} | {Metadata X/50, Visibility X/25, Conversion X/25} |
   ```

2. **Update Top Findings** instruction:
   - Change from "List the 3 most impactful findings" to: "List the top 3 recommendations from the Prioritized Recommendations section above, summarized in one line each with their impact level."

3. **Replace the Next Steps placeholder:**
   - Remove: `"For scoring, competitive context, and a full report file, these features will be available in a future update."`
   - Replace with: `"For a full report file saved to your working directory, this feature will be available in a future update. To optimize your listing based on these findings, run /aso:optimize."`
