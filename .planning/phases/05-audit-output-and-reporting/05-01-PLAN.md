```
## Report Output

After completing the analysis and printing the Analysis Summary above, save the full audit as a Markdown report file.

### Filename Convention

Build the filename from the app name and current date:

`{app_name_slug}-aso-audit-{YYYY-MM-DD}.md`

Where:
- `{app_name_slug}` is the app name lowercased with spaces replaced by hyphens, non-alphanumeric characters (except hyphens) removed, and consecutive hyphens collapsed to one (e.g., "My App: Pro Edition!" becomes `my-app-pro-edition`)
- `{YYYY-MM-DD}` is today's date

Example: `calm-aso-audit-2026-04-02.md`

### Report Contents

Use the **Write tool** to save the file to the current working directory. The report must contain ALL of the following sections from this audit, in order:

1. **Header** -- App name, platform, date, and overall ASO score with letter grade
2. **Quick Stats Table** -- The same table from Analysis Summary
3. **Title Analysis** -- Full title analysis output
4. **Subtitle / Short Description Analysis** -- Full subtitle or short description analysis output
5. **Description Analysis** -- Full description analysis output
6. **Keyword Field Guidance** (iOS only) -- Full keyword field guidance output
7. **Rating and Review Summary** -- Full rating and review output
8. **ASO Score** -- Score breakdown table, letter grade, and category sub-scores
9. **Competitive Context** -- Competitor table, observations, and positioning assessment (or fallback message if unavailable)
10. **Prioritized Recommendations** -- All recommendations in impact order
11. **Footer** -- "Generated by /aso:audit on {YYYY-MM-DD}"

Format the header as:

```markdown
# ASO Audit: {App Name}

**Platform:** {iOS / Android}
**Date:** {YYYY-MM-DD}
**ASO Score:** {score}/100 ({grade})
**Metadata:** {X}/50 | **Visibility:** {X}/25 | **Conversion:** {X}/25
```

### Post-Save Confirmation

After writing the file, print:

`Report saved: ./{filename}`
```
