**iOS Keyword Field Composition** (per OPTM-04):

This section applies ONLY to iOS. For Android, print: "iOS keyword field not applicable -- Google Play has no hidden keyword field. Keywords are integrated via the description (see Description Optimization below)."

Construction Rules -- reference rules/aso-domain.md iOS Keyword Field Rules (same rules used in keywords.md iOS Keyword Field String section):
1. Comma-separated, NO spaces after commas
2. Do NOT include words already in the app's Title or Subtitle (Apple auto-indexes all three together)
3. Prefer singular forms over plural (Apple indexes both from singular)
4. Do NOT include competitor brand names (policy violation risk)
5. Omit prepositions and articles ("the", "a", "for", "and")
6. Use every available character -- target 95-100 chars used

Composition Process:
1. Identify all target keywords NOT already covered by the optimized title and subtitle variants (use the recommended title and subtitle from the sections above).
2. Decompose multi-word keywords into individual words. Only include a word if not already covered by another entry in the field or by the title/subtitle.
3. Convert plurals to singular forms.
4. Strip filler words (prepositions, articles).
5. Prioritize by relevance score if available from a prior /aso:keywords report, otherwise by estimated importance to the app's core use case.
6. Pack keywords comma-separated until approaching 100 chars.

Output:
- Display the composed keyword string in a code block
- Show: "{N}/100 characters used ({remaining} remaining)" (per OPTM-06)
- If over 100 chars: trim from lowest-priority end and note what was removed
- If under 90 chars: note available space for additional terms

Coverage Summary:
- Count unique searchable terms from combined title + subtitle + keyword field (deduplicated)
- List any high-priority target keywords that could not fit
- Suggest swaps if critical keywords were excluded (remove low-priority term X, add high-priority term Y)

Label all output as [ESTIMATED].

**Description Optimization** (per OPTM-05):

Branch by platform (same pattern as audit.md Description Analysis):

iOS Description (4,000 chars, NOT indexed):
Since Apple does NOT index descriptions for search, optimize purely for conversion:
1. Analyze current description: character count, first-3-lines quality, CTA presence, social proof.
2. Generate optimized description copy focusing on:
   - Strong opening hook in first 3 lines (the visible portion before "more" tap)
   - Benefit-focused language over feature-focused ("Track your sleep and wake refreshed" not "Sleep tracking feature")
   - Scannable formatting: short paragraphs, bullet points, visual breaks
   - Clear call-to-action at the end
   - Social proof if available (awards, user count, press mentions)
3. Present the optimized description with character count: "{N}/4,000 characters ({remaining} remaining)" (per OPTM-06)
4. Note key changes from the original and why each change improves conversion.

Google Play Description (4,000 chars, IS indexed):
Optimize for both keyword integration AND readability:
1. Analyze current description: character count, keyword density, keyword placement.
2. Generate optimized description copy focusing on:
   - Front-load primary keywords in first paragraph (highest ranking weight)
   - Target density: approximately 1 keyword mention per 250 characters
   - Natural keyword integration (readable sentences, not keyword stuffing)
   - Each priority keyword appearing 1-3 times (primary keyword up to 3x, others 1-2x)
   - Keywords distributed across the full description (not clustered)
   - Strong opening hook and closing CTA (same conversion value as iOS)
   - Scannable formatting with short paragraphs and bullet points
3. Present the optimized description with:
   - Character count: "{N}/4,000 characters ({remaining} remaining)" (per OPTM-06)
   - Keyword density report: "{keyword_count} keyword instances in {char_count} chars = 1 per {chars_per_keyword} chars"
   - List of target keywords placed and their frequency in the optimized text
4. Note key changes from the original.
