**YAML Frontmatter** (per OPTM-01):
- name: aso:optimize
- description: Generate optimized metadata variants for app store listings. Produces title, subtitle, keyword field, and description alternatives respecting platform character limits and ASO best practices.
- allowed-tools: WebSearch, WebFetch, Read, Write, Bash
- argument-hint: "&lt;current-metadata-or-keywords&gt;"
- No Agent tool needed (optimize does not spawn subagents)
- No disable-model-invocation (let Claude auto-invoke when user says "optimize my listing")

**Role Statement:**
"You are the ASO Metadata Optimization command. You generate optimized metadata variants for every text field in an app store listing, respecting platform character limits and ASO best practices defined in rules/aso-domain.md."
State that this command covers title, subtitle/short description, iOS keyword field, and description optimization. Instruct to read rules/aso-domain.md before starting.

**Input Resolution** (per OPTM-01) -- 3 paths matching audit.md/keywords.md pattern:

Path A - Explicit Argument: If $ARGUMENTS provided, parse for:
  1. Current metadata (app name/title, subtitle, description -- pasted inline or as a file path). Use the Read tool if a file path is detected.
  2. Target keywords (comma-separated list, or reference to a prior /aso:keywords report file). If a file path ending in keyword-research-*.md is detected, read it and extract the Prioritized Focus Keywords table.
  3. Store URL or app ID (optional) for platform detection using rules/aso-domain.md patterns.

Path B - Active App Context: If no $ARGUMENTS or only keywords provided without metadata, check for .aso-context.json in the working directory.
  - File exists: Load title, subtitle, short_description, description, platform, category. Print "Using active app: {app_name}" and proceed.
  - File missing: Fall through to Path C.

Path C - No Input: Ask user: "Provide your current metadata (title, subtitle, description) and target keywords. Or run /aso:app-new first to set up app context."

**Required Data After Resolution:**
- platform (ios or android) -- REQUIRED for character limits and field-specific logic
- current title -- REQUIRED (the text to optimize)
- target keywords -- REQUIRED (what keywords to incorporate)
- current subtitle (iOS) or short_description (Android) -- used if available, optimized even if missing (treat as greenfield)
- current description -- used if available for description optimization (Plan 02)

Post-resolution confirmation: "Optimizing metadata for: {app_name} ({platform}) | Target keywords: {count} keywords"

**Title Optimization** (per OPTM-02):

Section header with ranking weight context (same pattern as audit.md): "Title carries the highest weight in both iOS and Google Play ranking algorithms."

Instructions:
1. Analyze current title: report character count vs platform limit (30 chars for both platforms per rules/aso-domain.md), identify keywords present, note keyword positioning.
2. Generate 3-5 title variants. Each variant MUST:
   - Respect the platform character limit (30 chars iOS, 30 chars Google Play)
   - Front-load the highest-value target keyword (first position or first phrase)
   - Maintain brand identity (keep brand name if present in current title)
   - Read naturally (no keyword-stuffed gibberish)
   - For Google Play: avoid prohibited words from rules/aso-domain.md (no "Best", "#1", "Free", "Top", ALL CAPS, emojis)
3. Present variants in a table:

| # | Variant | Chars | Keywords Covered | Strategy |
|---|---------|-------|-----------------|----------|
| 1 | {variant text} | {N}/30 | {keyword1, keyword2} | Keyword-first with brand |
| 2 | ... | ... | ... | Brand-first with keyword |
| ... | ... | ... | ... | ... |

4. For each variant, show: "{N}/30 characters ({remaining} remaining)" (per OPTM-06)
5. After the table, provide a brief recommendation noting which variant best balances keyword coverage with brand recognition and readability.
6. Label all variant analysis as [ESTIMATED].

**Subtitle / Short Description Optimization** (per OPTM-03):

Branch by platform (same pattern as audit.md):

iOS Subtitle (30 chars):
1. Analyze current subtitle: character count vs 30-char limit, keywords present, overlap with title.
2. Generate 3-5 subtitle variants. Each variant MUST:
   - Respect 30-char limit
   - NOT duplicate any word already in the chosen/current title (Apple auto-indexes title + subtitle together, duplication wastes space)
   - Cover high-value target keywords not already in the title
   - Communicate a clear value proposition
3. Present in table format (same as title) with character count and keywords covered.
4. Flag any variant that duplicates a title word: "WARNING: '{word}' duplicates title -- wasted indexing space"
5. Show remaining character count for each variant (per OPTM-06).

Google Play Short Description (80 chars):
1. Analyze current short description: character count vs 80-char limit, keywords present.
2. Generate 3-5 short description variants. Each variant MUST:
   - Respect 80-char limit
   - Integrate target keywords naturally (this field IS indexed on Google Play)
   - Communicate the core app value in one scannable line
3. Present in table format with character count and keywords covered.
4. Show remaining character count for each variant (per OPTM-06).

Missing Field: If subtitle/short description is empty or missing, note this as an opportunity and generate 3-5 options from scratch.

**Data Labeling Convention** section at the end (same as audit.md and keywords.md):
- OBSERVED: Current metadata character counts, existing text
- ESTIMATED: Optimization variants, keyword coverage assessments, strategic recommendations

Do NOT include: iOS keyword field composition, description optimization, before/after comparison, keyword coverage verification, or report output sections. Those belong to Plan 02 (keyword field + description) and Phase 9 (validation + report).
