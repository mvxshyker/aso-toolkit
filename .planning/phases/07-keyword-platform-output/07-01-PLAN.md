**Section 1: "## iOS Keyword Field String" (KWRD-06)**

This section applies ONLY to iOS apps. For Android apps, print: "iOS keyword field not applicable -- see Google Play Keyword Integration below." and skip to the next section.

Subsections:

1. **Construction Rules** -- Reference rules/aso-domain.md iOS Keyword Field Rules directly. Restate the rules briefly inline so the executor does not need to cross-reference during generation:
   - Comma-separated, NO spaces after commas (spaces consume characters)
   - Do NOT include words already in the app's Title or Subtitle (Apple auto-indexes title+subtitle+keyword field together)
   - Prefer singular forms over plural (Apple indexes both from singular)
   - Do NOT include competitor brand names (policy violation risk)
   - Omit prepositions and articles ("the", "a", "for", "and") -- they waste space
   - Use every available character -- unused space is wasted ranking opportunity

2. **Keyword Selection** -- From the scored keyword list (Relevance Scoring section output), select keywords for the field using this priority:
   - Start with relevance 8-10 keywords NOT already in the title or subtitle
   - Then relevance 6-7 keywords NOT already in the title or subtitle
   - If user provided volume data [USER-PROVIDED], use volume as a tiebreaker within the same relevance tier (higher volume first)
   - If popularity tiers are available [ESTIMATED], prefer High/Medium over Low/Unknown within the same relevance tier
   - Convert multi-word phrases to individual words (e.g., "guided meditation" contributes "guided" and "meditation" as separate words -- only include a word if it is not already covered by another keyword or the title/subtitle)
   - Convert plurals to singular forms
   - Strip prepositions, articles, and filler words

3. **Generated Keyword String** -- Build and display the keyword field string:
   - Format: `keyword1,keyword2,keyword3,related,term,synonym` (no spaces after commas)
   - Show character count: `{N}/100 characters used ({remaining} remaining)`
   - If the string exceeds 100 characters, trim lowest-priority keywords until it fits
   - If the string is under 90 characters, note: "Consider adding more terms to maximize coverage -- {remaining} characters available."

4. **Coverage Report** -- After the string, show what the combined title + subtitle + keyword field covers:
   - List how many unique searchable terms the combined metadata produces
   - Identify any high-relevance (8+) keywords that could NOT fit in the keyword field
   - If critical keywords were excluded, note them as "Overflow -- consider swapping with lower-priority terms in the keyword field"

Label the entire section: `Keyword field string is an optimized recommendation based on relevance analysis and formatting rules. [ESTIMATED]`

**Section 2: "## Google Play Keyword Integration" (KWRD-07)**

This section applies ONLY to Google Play (Android) apps. For iOS apps, print: "Google Play integration not applicable -- see iOS Keyword Field String above." and skip to the next section.

Subsections:

1. **Integration Strategy** -- Google Play indexes the full description for search. Unlike iOS (which has a dedicated keyword field), Android keyword optimization happens by naturally weaving keywords into the description text. Reference the benchmark from rules/aso-domain.md: approximately 1 exact keyword match per 250 characters of description.

2. **Priority Keywords for Description** -- From the scored keyword list, select the top 15-20 keywords (relevance 6+) that should appear in the description. Present as a table:

   | Priority | Keyword | Relevance | Suggested Placement |
   |----------|---------|-----------|-------------------|
   | 1 | {keyword} | {X}/10 | First paragraph (front-load) |
   | 2 | {keyword} | {X}/10 | First paragraph |
   | ... | ... | ... | Feature list / mid-description / closing |

   Placement guidance:
   - Relevance 9-10: First paragraph (front-loaded for maximum ranking weight)
   - Relevance 7-8: Feature descriptions or mid-description paragraphs
   - Relevance 6: Closing section or secondary feature mentions

   If user provided volume data, include it as a column and factor it into priority ordering.

3. **Title and Short Description Keywords** -- Separately list the top 3-5 keywords that should appear in the title (30-char limit) and short description (80-char limit). These are the highest-relevance, most competitive keywords. Note: title and short description keywords should also appear in the full description for reinforcement.

4. **Density Guidance** -- For a typical 2,000-4,000 character description:
   - Target: 8-16 keyword instances across the full description
   - Each priority keyword should appear 1-3 times (once is sufficient, twice for primary terms, three times maximum for the single most important keyword)
   - Keywords must read naturally within sentences -- never keyword-stuff

Label: `Keyword integration guidance is based on relevance analysis and Google Play indexing behavior. [ESTIMATED]`

**Section 3: "## Prioritized Focus Keywords" (KWRD-08)**

This section applies to BOTH platforms. It synthesizes the analysis into a clear "here are your top keywords" list.

1. **Selection Criteria** -- Combine relevance score with available data to produce a priority ranking:
   - Primary sort: Relevance score (highest first)
   - If user provided volume [USER-PROVIDED]: Secondary sort by volume (higher first)
   - If popularity tiers [ESTIMATED] available (no user volume): Secondary sort by tier (High > Medium > Low > Unknown)
   - Tiebreaker: Feature-Seeking and Problem-Solving intent keywords rank above Category and Comparison keywords (higher conversion intent)

2. **Top 10-15 Focus Keywords** -- Present the prioritized list:

   | Rank | Keyword | Relevance | {Volume [USER-PROVIDED] or Popularity [ESTIMATED]} | Intent | Why Focus |
   |------|---------|-----------|-----------------------------------------------------|--------|-----------|
   | 1 | {keyword} | {X}/10 | {value} | {intent category} | {One sentence: why this keyword deserves focus -- combines relevance reasoning with competitive/volume signal} |
   | ... | ... | ... | ... | ... | ... |

   Select 10-15 keywords. If the total keyword list has fewer than 15 high-quality candidates (relevance 6+), include fewer rather than padding with weak keywords.

3. **Strategic Summary** -- After the table, provide a 2-3 sentence strategic overview:
   - Which intent categories dominate the focus list
   - Whether the list skews toward competitive (high-volume/high-relevance) or niche (lower-volume but high-relevance) keywords
   - One actionable recommendation for keyword strategy refinement
